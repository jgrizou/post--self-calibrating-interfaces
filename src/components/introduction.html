An interface sits between a user and the machine they want to control. Its purpose is to convey the user's intent to the machine, which can then act accordingly.

Think about the remote control of your TV. It is just a plastic box with buttons. But buttons are to be pressed and the letters printed on them indicate what they do. The physical action of pressing the button printed with '1' is transcribed into an electronic signal that is understood by the TV to display channel 1.

This ability for a machine to translate user's actions into their meaningsTo known what the user wants the machine to do when performing a particular action, like pressing a button to change channel. seems paramount for an interface to work. How could a machine respond to some commands if it does not know what those commands mean?

We showcase such a machine below. The plan is as follow:

  1. We will first present the interface, a PIN-entering device, and learn to type a PIN by pressing buttons of known meaningsIt will be clear to both you and the machine what each button is used for.. We will analyse this interface and highlight the various components that are at play.
  2. Fun will start right after, buttons will loose their markings and you will get to decide what each button means and use them that way. The machine won't know that but will nonetheless figure out your PIN, as well as the meaning of each button. We will decipher how it works and refine our analysis to include what we learned.
  3. Then things will turn wild, we will no longer use buttonsWhich are discrete/symbolic signals. but place points on a map, draw sketches and even use spoken commands. Once again, you will get to decide what map, drawings, or spoken commands you want to use. This will teach us about scaling this work to continuous domains, a challenging task requiring to learn a classifierOne that translates continuous user signals into their discrete meanings. without known labels.
  4. Finally, we will zoom out, present previous more formal work on the subject, highlight some remaining challenges, and ask how this work can be useful with concrete examples from brain-computer interfaces to the study of language formation.
